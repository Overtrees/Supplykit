# BBCC 送仓模式全链路测试

## 一、BBCC 与 传统模式对比

| 维度 | 传统 8RDC | BBCC |
|:---|:---|:---|
| 商家送货 | 送到 8 个 RDC 大仓 | 送到 1 个 B 仓（集货中转） |
| 仓库层级 | RDC（8个）→ 消费者 | B 仓（1个）→ C 仓（60+个）→ 消费者 |
| 物流接力 | 商家直送各 RDC | 商家→B仓 → 京东越库/调拨→C仓 |
| 商家物流成本 | 分送 8 个仓，零散运费高 | 集中送 1 个 B 仓，批量运费低 |
| B 仓存储费 | 无此环节 | 15 天免费，超期收费 |
| 库存可见性 | 8 个 RDC 独立库存 | B 仓存量 + 60+ C 仓分布 |

## 二、库存模型调整

### 传统模式
```
库存分布: 沈阳100/北京80/上海60/广州50/成都40/武汉40/西安30/郑州30 = 480
各仓各自计算: 安全线 = 仓日销 × 15天
补货触发: 任一仓 ≤ 安全线 → 向该仓补货
```

### BBCC 模式
```
库存分布: B仓(356) → 调拨 → C仓1/C仓2/.../C仓60+
B 仓负责: 接收、暂存、分拨（15天免费存储）
C 仓负责: 消费者发货（保持 3-5 天安全库存）
补货触发: B 仓存量 ≤ 再订货点 → 向供应商下单
C 仓补货: B 仓自动调拨（京东物流负责，1-3 天到达）
```

### 关键参数

| 参数 | 传统模式 | BBCC 模式 |
|:---|:---:|:---:|
| 商家送货目的地 | 8 个 | 1 个 |
| 物流时效（商家→前端） | 4-5 天 | 4-7 天（生产）+ 1-3 天（B→C）= 5-10 天 |
| 前端最低周转 | 15 天（8 仓各自） | 3-5 天（C 仓，B 仓自动补） |
| B 仓存储上限 | — | 15 天（超期收费） |
| 再订货点 | 日销 × (15+5) | 日销 × (生产7+补货周期3+安全5) = 日销 × 15 |

## 三、BBCC 模式补货计算

### 参数设定

| 项目 | 值 | 说明 |
|:---|:---:|:---|
| 生产+到 B 仓 | 7 天 | 含生产周期 4-7 天 |
| B→C 调拨 | 3 天 | 越库/调拨到全国 C 仓 |
| C 仓安全库存 | 5 天 | 确保不断货 |
| B 仓免费存储 | 15 天 | 超期收仓储费 |
| **再订货点** | **15 天** | 日销 × (7+3+5) |
| B 仓库存上限 | 15 天 | 日销 × 15，超过即产生费用 |

### 计算逻辑

```
再订货点 = 日均销 × (生产周期 + B→C调拨 + C仓安全)
         = 日均销 × (7 + 3 + 5) = 日均销 × 15天

建议采购量 = 日销 × (B仓上限 + B→C调拨周期) - B仓库存 - 在途
           = 日销 × (15 + 3) - B仓库存 - 在途
           = 日销 × 18 - B仓库存 - 在途

B仓周转 = B仓库存 / 日均销
若 B仓周转 > 15天 → 需加快调拨至C仓，避免仓储费
若 B仓周转 < 5天  → 触发紧急补货
```

## 六、BBCC vs 传统 模式对比总结

### 对商家端的影响

| 项目 | 传统 8RDC | BBCC | 差异 |
|:---|:---|:---|:---:|
| 送货次数 | 8 次（每仓 1 次） | 1 次（送 B 仓） | **减少 7 次** |
| 运输复杂度 | 高（多仓多车次） | 低（集中发运） | ✅ 简化 |
| B 仓仓储费 | 无此费用 | 超 15 天收费 | ⚠️ 需控制 B 仓库存周转 |
| 库存水位 | 各仓独立备货，总量冗余 | B 仓集中，C 仓动态调拨 | ✅ 总量更低 |
| 缺货风险 | 一个仓缺货，其他仓可能有 | B 仓集中调配，灵活度高 | ✅ 风险分散 |
| 系统适配 | 按仓库维度计算补货 | B 仓+调拨节奏两维度 | ⚠️ 需要调拨逻辑 |

### 系统改造建议

当前系统架构基本支持 BBCC，仅需调整：

1. **「前置仓」概念重建**：RDC → C 仓，库存维度从 8 个仓变为 B 仓+60+C 仓
2. **补货公式更新**：增加 `b_to_c_days` 和 `c_safety_days` 两个参数
3. **B 仓周转告警**：当 B 仓库存 > 日销 × 15天 时，告警「仓储费风险」
4. **调拨建议输出**：除供应商采购单外，增加「B→C 调拨建议」，指导京东物流分配

## 七、测试结论

**全链路通畅 ✅**

BBCC 模式对系统的改动量比预期小——核心变化不在导入/存储/统计层，而在**补货计算逻辑**：
- 安全线公式不变，参数从 8 仓各自取值改为 B 仓统一取值
- 新增「B 仓周转告警」（超 15 天免费期）
- 导出采购单从「分仓采购」改为「送 B 仓统一采购」
- 特殊时刻备货更简单（集中送 B 仓，由京东分拨）

系统当前架构兼容两种模式，切换成本主要在后端补货公式，前端无需改动。
